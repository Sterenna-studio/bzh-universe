# Statuts canon et usage

Cette page fixe les libelles a utiliser dans le wiki quand un element n'a pas le meme niveau de validation qu'un autre. Le but est de garder les idees, sources et references visibles sans les confondre avec le canon final.

## Statuts principaux

| Statut | Usage | Decision pratique |
| --- | --- | --- |
| `canon` | Element valide comme faisant partie de l'univers ou du projet actif. | Peut etre relie depuis les pages pivots sans reserve. |
| `a confirmer` | Element coherent mais pas encore arbitre. | Le garder visible, mais signaler l'incertitude. |
| `piste` | Idee, variante, prompt, concept ou direction possible. | Ne pas l'utiliser comme fait de lore. |
| `reference` | Media ou document utile pour l'inspiration, la direction artistique ou la production. | Ne pas le traiter comme asset final. |
| `archive` | Trace historique conservee pour provenance. | Peut etre citee, mais pas reprise sans verification. |
| `interne` | Contenu conserve hors navigation publique directe. | Ne pas le linker depuis les pages de navigation publiques. |
| `source brute` | Fichier conserve tel quel avant synthese ou classement. | Le decrire par provenance et ne pas le renommer sans tri. |

## Regle de nom lisible

Les fichiers historiques peuvent garder leur nom original. Quand un nom est trop brut, ajouter dans le wiki un titre humain court, par exemple :

| Chemin brut | Nom lisible | Statut |
| --- | --- | --- |
| `media/video/references/desktop-bzh/son/desktop-bzh-son-mutenrock-v002.mp4` | Lyric video MutenRock v2 | `reference` |
| `media/video/references/desktop-bzh/chibi-bzh-power-decembre-2024/` | Clips chibi BZH POWER decembre 2024 | `reference` |
| `assets/cards/bzh01/cards/` | Set de cartes BZH01 | `canon` / `a confirmer` selon la carte |

## Application par zone

- Pages personnages : indiquer le statut des alias, roles, musiques, cartes et references visuelles.
- Pages media : separer `master`, `reference`, `archive` et `a confirmer`.
- Pages projets : distinguer projet actif, prototype, archive et piste.
- Imports : ne promouvoir un fichier dans le wiki public que si son statut est explicite.
- Archives internes : garder une trace de leur existence, sans lien direct si le contenu est marque `interne`.
